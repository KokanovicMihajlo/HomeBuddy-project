# 10 Detailed Test Cases for HomeBuddy Application

This document provides 10 detailed test cases for the HomeBuddy application, covering various functionalities and scenarios, including the newly accessible multi-step wizard.

| Test Case ID | Title | Preconditions | Steps | Expected Results |
|---|---|---|---|---|
| **TC-001** | Verify Landing Page Loads Successfully (Smoke) | User has a stable internet connection and a supported web browser. | 1. Open the browser. <br> 2. Navigate to https://hb-test.stage.sirenltd.dev/ | The landing page loads successfully, and the main heading "Complete Kitchen Remodeling" is visible. All critical elements like the ZIP code input and "Get estimate" button are displayed. |
| **TC-002** | Verify "Get estimate" with a valid ZIP code that has no contractors (Negative) | User is on the landing page. | 1. Enter a valid ZIP code (e.g., 46220) in the ZIP code input field. <br> 2. Click the "Get estimate" button. | The user is redirected to a page with the message "Unfortunately, I have no matching contractors in your area yet." |
| **TC-003** | Verify "Get estimate" with an empty ZIP code (Negative) | User is on the landing page. | 1. Leave the ZIP code input field empty. <br> 2. Click the "Get estimate" button. | An inline validation error message should appear, indicating that the ZIP code is required. The page should not reload or redirect. |
| **TC-004** | Verify "Get estimate" with a non-numeric ZIP code (Negative) | User is on the landing page. | 1. Enter non-numeric characters (e.g., "ABCDE") into the ZIP code input field. <br> 2. Click the "Get estimate" button. | The input field should either prevent the entry of non-numeric characters or display a validation error message upon submission. |
| **TC-005** | Verify successful completion of the multi-step wizard (Happy Path) | User is on the landing page. | 1. Enter ZIP code **10001** and click "Get estimate". <br> 2. Select **Kitchen cabinets** and **Countertop(s)**. <br> 3. Select **Replace all or most cabinets**. <br> 4. Select **Single family home**. <br> 5. Select **No** (for mobile home). <br> 6. Select **Yes** (for homeowner). <br> 7. Enter **150** sq. ft. <br> 8. Select **$10K to $30K** budget. <br> 9. Enter **John Doe** and **john.doe@example.com**. <br> 10. Enter phone number **2025550123** and confirm. | The user is successfully redirected to the "Thank You" page with a confirmation message. |
| **TC-006** | Verify personalization on the "Thank You" page (Functional) | User has successfully completed the wizard (TC-005). | 1. Observe the confirmation message on the "Thank You" page. | The confirmation message should use the name entered in the contact step (e.g., "Thank you **John Doe**..."). |
| **TC-007** | Verify phone number validation (Negative) | User is on the phone number step of the wizard. | 1. Enter a phone number with less than 10 digits (e.g., 12345). <br> 2. Click "Submit my request". | An inline validation error message should appear, indicating that the phone number must be 10 digits long. |
| **TC-008** | Verify "Terms of Use" link navigation (Functional) | User is on the landing page. | 1. Scroll to the footer of the page. <br> 2. Click on the "Terms Of Use" link. | The user should be redirected to the correct "Terms of Use" page. |
| **TC-009** | Verify email submission with an invalid email on "no contractors" page (Negative) | User is on the "no matching contractors" page. | 1. Enter an invalid email address (e.g., "invalid-email") in the email input field. <br> 2. Click the "Submit" button. | An inline validation error message "Wrong email format" should appear. |
| **TC-010** | Verify the "Back" button functionality in the wizard | User is on the "What type of property is this?" step of the wizard. | 1. Click the "Back" button. | The user is successfully navigated back to the previous step ("What would you like to do with your kitchen cabinets?"), and the previous selection is maintained. || **TC-011** | Verify wizard logic when non-cabinet items are selected | User is on the landing page. | 1. Enter ZIP code **10001** and click "Get estimate". <br> 2. On the "Which elements of the kitchen would you like to update?" step, select **Countertop(s)** and **Flooring** (do NOT select Kitchen cabinets). <br> 3. Click the "Next" button. <br> 4. A pop-up message appears: "Our contractors only do projects including kitchen cabinets. Would you still like to continue?". Click "Yes". | The wizard should either adapt to the user\'s selected elements (Countertop(s) and Flooring) and proceed with relevant questions for those selections, or if kitchen cabinets are mandatory, it should clearly state this upfront or prevent continuation without selecting cabinets. The wizard should NOT force a cabinet-focused flow after the user explicitly chose not to include cabinets. |
